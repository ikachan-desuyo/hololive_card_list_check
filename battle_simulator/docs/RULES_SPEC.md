## 1. ゲームの基本

- 2人対戦（1.1.1）
- 一方が敗北した時点でゲーム終了。敗北していない側が勝利（1.2.1）。両者同時敗北は引き分け（1.2.2）

### 敗北条件（実装必須・チェックタイミングで判定するルール処理）

| # | 条件 | 条番号 |
|---|------|--------|
| 1 | 自分のライフエリアにカードが無い | 11.2.1 |
| 2 | 自分のステージ（センター/コラボ/バック）にホロメンがいない | 11.2.2 |
| 3 | 手札ステップでカードを引く際にデッキが0枚 | 7.3.2.1 |
| 4 | ゲーム準備中の引き直し回数が6回に達した | 6.2.1.9.1 |
| 5 | 投了（即時、置換不可） | 1.2.3 |

- カード効果による勝利/敗北はチェックタイミングを待たず即時（1.2.4）

### 大原則

- カードテキストはルールより優先（1.3.1）
- 実行不可能な指示は単に行わない。一部のみ可能なら可能な範囲で実行（1.3.2）
- 「行動を指示する効果」と「禁止する効果」が衝突したら禁止が常に優先（1.3.3）
- 同時選択はターンプレイヤー（ゲーム開始前は先攻）から。後の選択者は先の選択内容を見られる（1.3.4）
- 選ぶ数は0以上の整数。「～まで」は0を選択可（1.3.5）

## 2. カード情報

### カードタイプ（2.2）

| タイプ | 識別 | 補足 |
|--------|------|------|
| 推しホロメン | ライフ情報を持つ | デッキ外。推しポジションに置く |
| ホロメン | HP情報を持つ | Bloomレベル: デビュー / ファースト / セカンド / スポット |
| サポート | 補助タイプあり | アイテム/イベント/スタッフ（使い切り）、ツール/マスコット/ファン（ホロメンに付ける） |
| エール | — | エールデッキ専用。テキストは注釈文扱い（2.7.2.1） |

### 色（2.4）

白・緑・赤・青・紫・黄 ＋ 無色。

- 無色は「色」として数えない（2.4.2.2）。コスト表記の無色は「任意の色1つ」（10.5.2）
- 多色カードは構成色すべてを持つ（2.4.3）。「同色」= 共通する色が1つでもある（2.4.4）

### その他の情報

- **HP**: 残りHP = カードのHP − 負っているダメージ（4.4.5.4）
- **バトンタッチ**: バトンタッチに必要なエールコスト（2.10）
- **Buzz**: `Buzz` 表記。ダウン時ライフ−2 等のエクストラ能力と組で使われる（2.5, 2.11.2.2）
- **LIMITED**: プレイ制限表記（ターン1枚・先攻1ターン目不可）（2.12, 8.6.2）
- **エクストラ能力**（2.11.2）: 「デッキに何枚でも入れられる」「ダウンした時ライフ-2」「〈名称〉としても扱う」「Bloomできない」
- **タグ**: ルール上の意味なし。効果からの参照のみ（2.6.2）

- **ステージ** = センター + コラボ + バックの総称。ゲーム開始時のステージ合計ホロメン上限は6（4.6.2.1）

### ホロメンの実体（4.4）— エンジン中核データ構造

ステージのホロメン1人 = **ホロメンカードのスタック + 付けられたカード（エール/サポート）+ 負っているダメージ**。

- 情報参照は常にスタック最上段のカードを基準（4.4.2.1）
- ステージ内移動（センター⇔コラボ⇔バック）では同一ホロメンとして状態・ダメージ・適用効果を引き継ぐ（4.1.4）
- ステージ外へ移動すると最上段のみ移動し、下のカードと付いているカードはアーカイブ（4.4.7）。移動先では「新しいカード」扱い（効果は引き継がない）
- 配置状態: 「お休みしていない（アクティブ）/ お休み」＋「表向き / 裏向き」（4.3）

## 5. ゲーム準備（6.2）— セットアップのステートマシン

1. デッキをデッキエリアへ置きシャッフル、エールデッキも同様
2. 推しホロメンを表向きで推しポジションへ
3. ランダムに1人を決め、そのプレイヤーが先攻/後攻を選ぶ
4. 各自デッキから7枚引く
5. **任意の引き直し**（先攻から、各自1ゲーム1回）: 手札を全てデッキに戻しシャッフル、7枚引き直す（6.2.1.7）
6. 引き直し回数 = 0 で初期化
7. **強制マリガン**（先攻から繰り返し）: 手札にデビューのホロメンが無ければ手札を公開→デッキに戻しシャッフル→7枚引く→引き直し回数+1。回数が6に達していたら敗北（6.2.1.9）
8. **初期配置**（先攻から）: デビュー1枚をセンターに裏向きで置ける。引き直し回数 ≥ 1 なら、その枚数分手札からカードを選びデッキの下へ（ペナルティ）。デビュー/スポットを任意枚数バックに裏向きで置ける（6.2.1.10）
9. エールデッキの上から推しのライフ値分をライフエリアへ裏向きで移動（6.2.1.11）
10. 裏向きのステージカードを全て表に（6.2.1.12）
11. 先攻プレイヤーのターンから開始

## 6. ターン進行（7）— フェーズのステートマシン

各ステップでチェックタイミング（CT、§9参照）が随所に入る。

### 6.1 リセットステップ（7.2）
- そのプレイヤーの最初のターンならスキップして手札ステップへ（7.2.1）
- 自ステージの全カードをアクティブにする
- コラボのホロメンをバックへ移動し、お休みにする（7.2.3）
- 「ターンの始めに」誘発 → CT
- センターが空なら、バックのアクティブなホロメン優先で1人をセンターへ（いなければお休みのホロメンから）。プレイヤーが選ぶ（7.2.6）→ CT

### 6.2 手札ステップ（7.3）
- 「手札ステップの始めに」誘発 → CT
- 1枚引く。**デッキ0枚で引けなければ敗北**（7.3.2.1）→ CT

### 6.3 エールステップ（7.4）
- 「エールステップの始めに」誘発 → CT
- エールデッキの一番上を公開し、自ステージの任意のホロメン1人に付ける（7.4.2）→ CT

### 6.4 メインステップ（7.5）
ターンプレイヤーは以下のアクションを任意の順で何回でも実行可（各実行ごとにCT）。完全に実行できないアクションは選択不可（8.1.3）。

| アクション | 制約 | 条番号 |
|-----------|------|--------|
| 手札のホロメンをバックに出す | デビュー/スポットのみ。ステージ数 < 上限(6) | 8.2 |
| Bloom | 自分の最初のターンは不可。対象: スポットでない / このターンに出ていない / このターンBloomしていない。手札のカードと対象が**同名**で、1st→(デビュー or 1st)、2nd→(1st or 2nd)、かつ新カードHP > 現在の累積ダメージ | 8.3 |
| コラボ | ターン1回。コラボが空いている時、バックの**アクティブな**ホロメン1人を選び、デッキ上1枚をホロパワーへ置き（デッキがある場合）、コラボへ移動 | 8.4 |
| 推しスキル / SP推しスキル | 推しスキルはターン1回、SPはゲーム1回。コスト=ホロパワーから表記枚数をアーカイブ | 8.5, 12.1.3 |
| サポートカードのプレイ | LIMITED: ターン1枚まで、かつ先攻の1ターン目は不可 | 8.6 |
| バトンタッチ | ターン1回。アクティブなセンターから、バトンタッチコスト分の付いているエールをアーカイブし、アクティブなバックホロメンと交代 | 8.7 |

- Bloom 時の処理（5.14）: スタック最上段に重ねる。ダメージ・状態・適用効果は引き継ぐが、テキスト/情報は新カードのものに変わる。同名能力でも「別の能力」扱い（回数制限はリセット）

### 6.5 パフォーマンスステップ（7.6）
- **先攻の第1ターンはスキップ**（7.6.1）
- センターホロメンとコラボホロメンが、それぞれ**各1回**アーツをプレイできる（9.1.1, 9.2.1.5）
- お休みしているホロメンは選べない（9.2.1.2）

アーツの解決（12.3.3–12.3.4）:
1. コスト判定 = アーツのコスト色がそのホロメンに付いているエールで満たされていること（**エールは消費されない**、無色コストは任意の色でよい）
2. 対象 = 相手のセンターかコラボのホロメン1人
3. 与ダメージ = 基本値 → テキスト効果による修正 → **特攻**（対象の色が特攻アイコンの色と一致なら加算）→ その他の加減算
4. 対象にダメージを与える（アーツダメージ）

### 6.6 エンドステップ（7.7）
- 「エンドステップの始めに」「ターンの終わりに」誘発 → CT
- 「ターンの終わりまで」「そのターン中」の効果が消滅（7.7.4）
- センターが空なら補充（アクティブ優先、バックが空ならコラボにいても移動せず空のまま）（7.7.5）
- 相手をターンプレイヤーとして次のターンへ

## 7. ダメージ・ダウン・ライフ（11）

### ダウン処理（11.3）— ルール処理
ホロメンの累積ダメージ ≥ HP、またはダウン状態の付与で:
1. 「ダウンした時」「ダウンさせた時」の能力を実行（ターンプレイヤー側から1つずつ選択して解決）
2. そのホロメンの全ホロメンカードをアーカイブし、コントローラーの**ライフダメージ +1**（Buzz等の置換で2になる場合あり: 2.11.2.2）
3. 複数同時ダウンはターンプレイヤーのホロメンから1人ずつ処理（11.3.2）

### ライフダメージ処理（11.5）— ルール処理
ライフダメージ ≥ 1 のプレイヤーは（他のルール処理がすべて済んでから）:
- ライフの一番上を公開し、**自分の任意のホロメンに送る**（エールとして付く）→ ライフダメージ −1
- ターンプレイヤーの処理が優先（11.5.1.1）

※ ライフが既に0枚なら敗北処理（11.2.1）が先に成立する。

### 回復・軽減（5.22, 5.23）
- 回復 = 累積ダメージを減らす（0未満にはならない）
- 軽減 = ダメージを与える際の置換（値を減じる）。「ダメージを受けない」= 全量軽減

### 不正カード処理（11.4）— ルール処理
- ホロメンに関連付いていたカードが宙に浮いたらアーカイブ
- 付け上限超過（ツール1/マスコット1 per ホロメン: 5.17.3）は上限分を選んで残し他をアーカイブ

## 8. 能力と効果（10）

### 能力の4分類（10.1）
| 種別 | 発動 | 表記 |
|------|------|------|
| 起動能力 | プレイタイミングにコストを払って能動的に | `[条件] コスト：効果` |
| 自動能力 | 誘発条件を満たすと待機状態になり、CTで強制プレイ | `（条件）した時、（効果）` |
| 永続能力 | プレイされず常時有効 | `（効果）` |
| サポート能力 | サポートカードのテキスト | |

- ホロメンの能力は「ステージにあり、かつスタック最上段」の間のみ有効（10.3.4.1）
- 自動能力は強制（任意コスト付きを除く）。複数待機時はコントローラーが順序を選ぶ（10.8.3）
- キーワード能力（13）: **コラボエフェクト** = コラボした時の自動能力 / **ブルームエフェクト** = このカードでBloomした時の自動能力 / **ギフト**

### 効果の3分類（10.2）
- 単発効果 / 継続効果（期限あり or ゲーム中）/ 置換効果（「かわりに」）
- 継続効果の適用順: 素の情報 → 能力の付与・無効化 → 数値以外の変更 → 数値変更 → 依存関係順 → 発生順（10.10.1）
- 置換効果は同一事象に各1回まで。「与える」置換が「受ける」置換より先（10.11.2）

### サポートカードの解決（10.7.2.5.1）
- アイテム/イベント/スタッフ → テキスト実行後アーカイブ
- ツール/マスコット/ファン → 選んだホロメンに付ける（上限超過なら付けられずアーカイブ）
- 「アーカイブされる以外にゲームの状態が変化しない」サポートはプレイ不可（10.7.2.3.7）

## 9. チェックタイミング（CT）とプレイタイミング（10.6）— エンジンのメインループ

```
checkTiming():
  loop:
    1. 条件を満たす全ルール処理を同時に実行（敗北判定→ダウン処理→不正カード→ライフダメージ処理）
       新たなルール処理が発生する限り繰り返す
    2. ターンプレイヤーの待機中自動能力があれば1つ選んでプレイ・解決 → 1へ戻る
    3. 非ターンプレイヤーの待機中自動能力があれば1つ選んでプレイ・解決 → 1へ戻る
    4. どちらも無ければCT終了

playTiming(player):
  checkTiming()
  loop:
    player が可能な行動を1つ実行 or パス
    行動したら checkTiming() を挟んで再びプレイタイミング
    パスしたらステップ進行
```

- ルール処理は「CTの時点で条件を満たしているか」のみで判定（11.1.2）
- ゲームの全状態変化はこのループを通る。**新エンジンはこの2関数を中心に設計すること**

## 10. 推しスキル（12.1）

- 推しスキル: ターン1回（メインステップのアクションとして）。SP推しスキル: ゲーム1回
- コスト: ホロパワーエリアから表記枚数をアーカイブ。「X」は1以上の任意値（12.1.3.1）
- 「（条件）する時に使える：」は選択型置換効果、「ダウンした時に使える：」はダウン処理中に実行（12.1.5）

## 12. パフォーマンス処理の8段階パイプライン（ルールブック p23, p28-30）

アーツ1回の処理は以下の順で進む。カード効果の発動タイミングはこの段階に紐づけて実装する。

| 段階 | 処理 | ここで発動する能力表記 |
|------|------|------------------------|
| 1 | アーツを使用するホロメンの決定 | |
| 2 | 使用するアーツの選択 | 「アーツを使う時」 |
| 3 | 対象にする相手のホロメンの決定 | |
| 4 | アーツの能力（テキスト効果）の発揮 | 「サイコロを振る時/振った時」 |
| 5 | 特攻のチェック（対象の色一致なら加算） | |
| 6 | アーツダメージの数値の決定 | 「ダメージを受ける時」（軽減等の置換） |
| 7 | 対象へアーツダメージを与える | 「ダメージを受けた時」 |
| 8 | ダウン処理とライフ減少 | 「ダウンした時」「ダウンさせた時」 |

- 処理後、アーツを使えるホロメン（センター/コラボの未行動側）が残っていれば段階1へ戻る。いなければエンドステップへ（p23）
- ダメージ計算順: テキスト効果 → 特攻 → その他の加減算（12.3.4, p26）

## 13. 盤面レイアウト（プレイシート / ルールブック p8）— UI実装の正本

プレイヤーから見た自陣の配置（相手側は点対称）:

```
[ライフ]      [コラボ] [センター]   [推し]  [ホロパワー]
（左端・縦置き                              [デッキ]
  横向きカード）      ステージ
              [バック（最大エリア・下段）]
[エールデッキ]                              [アーカイブ]
（左下）                                    （右下）
```

| # | 場所 | 備考 |
|---|------|------|
| ① | ステージ | センター+コラボ+バックの総称。推し以外のホロメンを最大6人 |
| ② | 推しポジション | 推しホロメン1枚 |
| ③ | センターポジション | ホロメン1人 |
| ④ | コラボポジション | ホロメン1人 |
| ⑤ | バックポジション | 残りのホロメン |
| ⑥ | デッキ | 右列 |
| ⑦ | ホロパワー | 右列上（デッキの上隣）。コラボ時にデッキ上から1枚置く |
| ⑧ | ライフ | 左端に横向きで縦に並べる |
| ⑨ | エールデッキ | 左下 |
| ⑩ | アーカイブ | 右下 |

- プレイシートにはターン進行（1.リセット〜6.エンド）の早見表も印刷されている

## 14. カードテキストの正本（プレイヤーズガイド）

- すべてのカードは**公式サイトのカードリストに掲載されているテキスト**のカードとして扱う（旧版の印刷テキストではなく）
- → カード効果実装時は `json_file/card_data.json` のテキストが公式リストと一致していることが前提。エラッタ情報は /rules/errata/ を確認

## 15. 実装上の注意（仕様の落とし穴）

- **エールはアーツのコストで消費されない**（12.3.3.1.1）。消費されるのはバトンタッチ（8.7.2）と効果による指定時のみ
- コラボ時のホロパワー置きは「デッキにカードがある場合」のみ必須。デッキ0でもコラボ自体は可能（8.4.3）
- Bloom の HP 条件は「新カードのHP > 累積ダメージ」。**以上ではなく超過**（8.3.3）
- リセットステップのコラボ→バック移動は「移動してからお休み」（7.2.3）。リセットでアクティブにした後に行われるため、コラボにいたホロメンはこのターンお休みのまま
- 先攻1ターン目: パフォーマンスなし・LIMITEDなし。Bloomは「各プレイヤーの最初のターン」不可（先攻1T・後攻1T両方）
- ステージ上限6はステージ全体（センター+コラボ+バック）の合計
- 手札からバックに出せるのはデビュー/スポットのみ。1st/2ndは必ずBloomで出す（8.2.2）
- 非公開領域の探索効果では「条件を満たすカードが無いことにできる」（4.1.2.3）— 対人検証は不要だがUI上「見つからなかった」を選べる必要がある
