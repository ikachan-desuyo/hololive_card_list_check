## A. 横断パターン（根因別）— 修正はここから着手するのが効率的

### P1. 多色ホロメン・別名・全色扱いの「生比較」（QA監査 A1 の広範囲再発）★最大
`color === '緑'` / `name === '…'` の完全一致判定が広く残存し、多色カード（白緑 SorAZ、青赤 FUWAMOCO 等）・
エクストラ「〈X〉としても扱う」（ラムダック等）・「全色扱い」継続効果が判定から漏れる。
エンジンには正しい `_hasColor()` / `_nameIs()` / `ctx.nameIs` が実装済みなのに使われていない。
- **hBD24-001〜067 のほぼ全数**（batch00: 実害33枚＋潜在17枚、batch01 も同様）
- hBP01-001 / 003 / 005 / 008 / 033（色）、hBP07-001（推し・ラムダック非対応）、hBP06-104 スバ友、
  hBP06-081 / 083（中）、hBP08-057 フワワ、hBP08-035 / 037（FUWAMOCO 赤→青エイリアス非対応・中）、hSD01-021 First Gravity（SorAZ）
- **text-compiler.js:150-156 も同根**（`e.top.color === color`。自動コンパイル分全体に波及する潜在）

### P2. 発動タイミング種別の取り違え（コラボ/ブルームエフェクト → 起動型能力）★悪用可能
コラボ時/Bloom時に1回だけ誘発すべき効果が、メインステップに**回数無制限で使える起動型能力**として実装されている。
- コラボエフェクト→起動型: hBP01-077 すいせい / hBP03-017 ぼたん / hBP03-062 ころね / hBP05-019 イオフィ / hSD02-011 ミオ
- ブルームエフェクト→起動型: hBP05-027 アキ / hBP05-034 ポルカ
- コラボエフェクト→ブルームエフェクト: hBP05-022 イオフィ

### P3. 任意/強制の逆転
- 強制効果が任意化（スキップできてしまう）: hBP03-070 / hBP03-083 / hBP04-007(SP) / hBP04-085(中) / hBP06-050 / hBP06-087 /
  hBP06-089 / hBP06-096 / hBP07-004(中) / hBP08-029(中) / hSD03-003 / hSD12-004 / hSD12-007
- 任意効果（「できる」「送れる」）が強制化: hBP05-069 / hBP05-072 / hBP07-056（候補1件で自動確定）/ hBP03-083（送り先も誤り）
- **デッキサーチで「見つからなかったことにできる」(4.1.2.3) が無い強制取得: hBD24系の約20枚**（同サイクル内で optional 実装と分裂）

### P4. 「選んで公開して送る」（サーチ）を「上から1枚」で実装
エールデッキ/デッキからの**選択**であるべき効果がトップ固定・ランダム化している。
- hBP02-002 レイネ推し「HALU」 / hBP02-059 カリオペ「Soul Voice」（デッキ）/ hBP06-006 ムーナSP /
  hBP08-033 レイネ / hSD12-016 GEOW / hSD13-015 ラオーラ

### P5. 「特定のSP推しスキルを使っていたら」を `usedSpOshiSkillThisGame` だけで判定
推し・スキル名を確認しないため、**無関係な推しのSPスキル使用でも条件成立**する。
- hBP06-044 / hBP06-045 / hBP06-056 / hBP07-107 / hBP06-046(中) / hSD06-010（ターン→ゲーム近似・JSDocに既知と明記）
- 同族: hBP02-087 魔法のステッキ（スキル名限定なしで推しスキル上限+1）

### P6. 規約プリミティブ非経由によるトリガー・カウント欠落
CLAUDE.md の「共通処理は context.js のプリミティブを必ず使う」に反し、直接配列操作で関連トリガーが発火しない。
