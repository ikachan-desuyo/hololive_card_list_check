# カード効果 実装状況（v2）

バトルシミュレーター v2（`battle_simulator_v2/`）のカード効果実装状況を管理する。
**正本は実コード**（`cards/<番号>.js` と `core/effects/`）。本ファイルは俯瞰・進捗管理用のサマリ。

> 数値は `json_file/card_data.json` から算出（最終更新時点）。再集計コマンドは末尾「更新ルール」を参照。
> テキストコンパイラの自動実装カバレッジだけは JS 実行が必要なため、ブラウザの
> `battle_simulator_v2/tests/coverage.html` で計測する（この開発機は Node/ブラウザ無し）。

最終更新: 2026-06-13

## 1. 規模と全体像

|---|---:|---|
| ユニークカード総数 | 1,052 | — |

| └ 効果実装が必要 | **874** | 下記3層のいずれかで実装 |

「効果なし」の内訳はエンジンが標準処理する：アーツ基本ダメージ／特攻(tokkou)／バトンタッチ／Bloom／コラボ（ホロパワー獲得）など。

## 2. 実装方式（3層）

1. **エンジンネイティブ** … 効果なしカード（178枚）。何もしなくても動く。
2. **手書き定義** `cards/<番号>.js` ＋ `cards/index.js` 登録 … 固有・複雑効果。
3. **テキストコンパイラ** `core/effects/text-compiler.js` … 手書きが無いカードの効果文を自動実装。
   **スキル枠単位で全文を解釈できた枠のみ**実装（1文でも未知パターンが混じれば、その枠は未実装のまま＝安全側）。

優先順位: 手書き ＞ コンパイラ自動 ＞ 未実装（`TODO(効果未実装)` ログ）。

## 4. エンジン機構ステータス

### 実装済み
- アーツ解決パイプライン（基本値→テキスト効果`run`→特攻→装着/継続修正→受けダメージ修正→適用）
- 効果フック: `bloomEffect` / `collabEffect` / `arts.<名>.{dmgBonus,run}` / `support.{canUse,run}`
- 推しスキル `oshiSkill` / `spOshiSkill`（ホロパワーコスト）、**ダウン時推しスキル** `onDownOshiSkill`
- **ダウン時トリガー** `triggers.onDown`（`_processDown` でアーカイブ前に発火）
- **装着時トリガー** `triggers.onAttach`（手動の装着アクション／`ctx.attachSupportWithTrigger` で発火）
- 装着の常時修正 `attached.{artsPlus,hpPlus,specialDmgPlus,damageDelta}` ＋ `attachRule`
- ターン継続修正 `addTurnModifier`（**`amount` に関数を許可＝可変値を解決時に再評価**）
- **「ターンに1回」ヘルパー** `ctx.oncePerTurnUsed/markOncePerTurn`
- **起動型能力** `activatedAbilities[]`（メインステップの「[コスト]：[効果]」。ホロメン自身/装着カード）
- **相手をダウンさせた時** `triggers.onOpponentDown`（カード共通）＋ **アーツ固有 `arts.<名>.onDownDealt`**（「このアーツで〜時」）。アーツ解決時に順次発火・選択可
- **アーツ必要エール軽減** `artsCostReduceAura(src,target,engine)` ＋ ターン修正 `kind:'artCostReduce'`（`_effectiveArtCost`）
- **バトンタッチ必要エール軽減** ターン修正 `kind:'batonCostReduce'` ＋ 装着 `batonCostReduceAttached`（`_effectiveBatonCost`）
- **装着カードによるアーツ必要エール軽減** `artsCostReduceAttached(host,engine)`（◆Buzzで無色-1 等）
- **エールデッキscry** `ctx.lookTopCheerDeck(n)/cheerDeckToBottom/sendRevealedCheer`
- **アーツで与ダメージ時** `arts.<名>.onDamageDealt(ctx,dealt)`（ライフスティール等）
- **アーツ使用条件** `arts.<名>.canUse(ctx)`（条件未達なら選択肢に出ない）
- **Bloomした時** `triggers.onBloom`（装着カード等。ホストのBloomで発火）
- **ダウン時の減るライフ-N** `h.lifeReductionOnDown`（onDownで設定→ライフ計上時に反映）
- **コラボした時の傍観** `triggers.onCollab`（他ホロメンが反応。ctx.player.collab がコラボしたホロメン）
- **バトンタッチ移動時** `triggers.onBatonMove`（バックへ移動した時）
- **複数ターン継続修正** `mod.untilTurn`（「次の相手のターン終了まで」等）／コスト修正は負値で必要エール増加も可
- 特殊ダメージ `dealSpecialDamage`（特攻・装着の特殊ダメージ＋・受けダメージ軽減を加味）
- **常時アウラ** `auraArtsPlus/auraHpPlus/auraDamageDelta/auraSpecialDmgPlus`（別ホロメンを恒常強化/保護。「#0期生全員+30」「コラボが受けるダメージ-10」「受けない＝-100000」「〈X〉の特殊+20」等。`_auraSum` で味方ステージを走査）
- **アーツを使った時** `triggers.onArtsUse`（アーツ解決後に発火・選択可。ホロパワー操作/特殊ダメージ/エール送り等）
- **リアクティブ被ダメージ割り込み（汎用）** `_collectDamageResponders(target,kind,defIdx)` が①推しスキル `oshi.onDamageOshiSkill` ②対象の装着カード `onDamageReceivedReact` ③防御側ステージのホロメンギフト `onDamageReceivedReact` を集約し、防御側へ順に決定ポイントを提示する。**アーツダメージ（`_offerDamageOshiSkill`、同期）と特殊ダメージ（`ctx.dealSpecialDamage`、ジェネレータ内 yield）の両経路で発火**（hBP03-105 ファン-30 / hSD13-012 特殊シールド）
- **被ダメージのターン修正** `damageReceivedDelta` がターン修正 `kind:'damageReceivedDelta'`（ownerIdx/matchKind/match/amount）も合算。「このターン自分のバックは特殊ダメージを受けない」等（hSD13-012）
- **ダイス割り込み** `ctx.rollDice()` はジェネレータ。振った後に自分の装着カードの `onDiceRollReact` をコントローラーへ提示（「目を4として扱う」hBP06-103／「振り直す」hBP01-123）。`ctx.dealSpecialDamage` も同様にジェネレータ化済み（呼び出しは `yield* ctx.xxx(...)`）
- **ダメージ種別の伝播** `arts`/`special` をダメージパイプライン全体に通す。`damageReceivedDelta(h,zone,kind)`・`auraDamageDelta(src,target,zone,engine,kind)` が種別を受け取り、「特殊ダメージのみ無効」等を表現可能（hBP07-088）
- **相手パフォ終了時** `triggers.onOpponentPerformanceEnd`（相手のパフォーマンスステップ終了時に防御側へ発火。`ctx.lifeDecreasedThisPerf` でそのステップ中のライフ減少を判定。hBP03-083）
- **効果Bloom（手札から）** `ctx.runBloomEffect(def,card,holomem)`＋手札からのstack.unshift。通常のターン制限を迂回する効果Bloom（コラボ等）に使用（hBP07-050）
- **攻撃元条件つき被ダメージ修正** ダメージパイプラインに攻撃元ホロメン(attacker)を伝播。`auraDamageDelta(src,target,zone,engine,kind,attacker)`／`attached.damageDelta(...,attacker)` で「相手の1stから受けるアーツ-30」等を表現（hBP06-072。`src===target` の自己ギフトもこの経路）
- **アーツ対象拡張** `_targetHolomem(playerObj,{zone,index})` で center/collab/back を統一解決。ターン修正 `kind:'artTargetDamagedBack'`（match でホロメン限定）を持つホロメンのアーツは相手のHP減バックも対象にできる（hBP07-086）
- **お休み移動／リセット非アクティブ** `ctx.moveToBackRestedSkipReset(holomem)`（持ち主自動判定でバックへ移動＋rested＋`skipNextReset`）。リセットステップで `skipNextReset` 付きはアクティブにせずフラグ消費（hBP06-088）
- **相手の決定ポイント** `ctx.opponentChoosesHolomem({filter,title})` は request.player を相手にして相手に選ばせる。`ctx.moveToCollabOwner(holomem)`（コラボ扱いでない位置移動）と併用（hBP03-088 / hBP07-081コラボ）
- **アーツダメージのリダイレクト（複数対象）** `arts.<name>.redirectTargets(ctx)` が対象ホロメン配列を返すと、指定対象のかわりにそれらへ与える（特攻・被ダメージ修正・推しスキル割り込み・ダウン処理を対象ごとに実行）（hBP07-081アーツ：ギラファ付きで相手センター＋コラボ）
- **特殊Bloom（レベル無視）** `def.specialBloom(h,handCard,engine,ownerIdx)` が true ならメインのBloom候補に追加（レベル遷移条件のみ迂回）（hBP01-045：ライフ3以下でDebut→2nd）
- **再アーツ** ターン修正 `kind:'reArts'`（match・`used`フラグ）を持つホロメンは、使ったアーツ（`h.lastArtUsedIndex`）をもう1回だけ使える（`_performanceActions` が再提示、使用で消費）（hBP07-008）
- **タイミング誘発の推しスキル** `onDiceRollOshiSkill`（ダイス時。`_offerDiceReact` 内）／`onArtsUseOshiSkills`・`onDamageDealtOshiSkills`（攻撃時。`_offerTimingOshiSkills` がダメージ解決後に提示。配列で通常＋SP併記可。`ctx.attackInfo` 参照）（hBP02-005 / hBP01-007）
- **前衛移動ロック** `spOshiSkill` でターン修正 `kind:'cannotMoveFrontline'`（相手側・`untilTurn`で次相手ターン終了まで）を積み、バトンタッチ生成時に参照（hBP01-005 SP）
- **エールアーカイブのコスト置換** `ctx.archiveCheer(h,cheer,{ability})` をジェネレータ化。`opts.ability!==false` のとき装着カードの `cheerArchiveReplace`（「エール1枚のかわりにこのファンをアーカイブ」）を提示。バトンタッチ等のシステムコストは `ability:false` で対象外（hBP03-106）

### 標準パターンのブルーム/コラボ効果 → コンパイラに委譲
