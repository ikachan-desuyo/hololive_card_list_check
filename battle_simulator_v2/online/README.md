# オンライン対戦（フレンド対戦・β）

GitHub Pages（静的ホスティング）上で、Firebase Realtime Database を中継ハブにしてフレンド同士の1対1対戦＋観戦を行う仕組み。

## 仕組み（なぜ軽量か）

- エンジンが**完全決定論**（シード付き乱数のみ）なので、対局は `(両デッキ + seed + 手の列)` から一意に再現できる。
- よってオンラインの同期は **「手(apply ID)を順序付きで共有するだけ」**。各クライアントが同じ手を同じ順で自分のエンジンに適用＝全員同じ盤面になる。リプレイ機能の基盤(applied列)をそのまま流用している。
- 観戦者もフルエンジンを回す＝**両方の手札が見える**（CPU対戦の観戦と同じ）。途中入室は過去の手ログを再適用して追いつく。
- フレンド限定のため**隠匿はしない**（各クライアントのメモリには相手の手札も存在する。覗き対策はしていない）。

## セットアップ（無料・5分）

1. https://console.firebase.google.com/ で新規プロジェクトを作成。
2. 「Realtime Database」を作成（テストモードで開始でOK）。下記ルールを推奨：
   ```json
   {
     "rules": {
       "rooms": { ".read": true, ".write": true, "$room": { ".indexOn": ["seq"] } }
     }
   }
   ```
   ※ルームコードを知る誰でも読み書き可能。荒らし対策が要るなら後で制限を追加。
3. プロジェクト設定 → 「マイアプリ」で Web アプリを追加 → `firebaseConfig` を `online/firebase-config.js` の `FIREBASE_CONFIG` にコピペ（`databaseURL` を含めること）。
4. デッキ選択画面の「🌐 オンライン対戦」が有効化される（未設定なら「未設定」と表示され無効）。

## 使い方

- **ルームを作る**：名前を入れて「ルームを作る」→ 表示されたコードを相手に伝える（Discord等）。
- **参加**：コードを入れて「参加」。空いていれば対戦者(p2)、満員なら自動で観戦。
- **観戦**：コードを入れて「観戦」。両者の手札が見える。
- デッキは画面上部の「プレイヤー1 のデッキ」を自分のデッキとして使う。
- 対局中の設定パネルは **任意効果の発動確認・投了・退室** のみ（AI/先読み/速度/評価値は非表示）。

## 既知の制限（β）

- ホスト(p1)が両デッキ確認後に seed/先攻を決めて開始する。ホストのブラウザが落ちても、Firebase上のデータは残るので観戦/復帰は可能（要再読込）。
- 1手ごとに Firebase 往復するため、複数選択は1枚ずつ確定で少しもたつく（正しく同期はされる）。
- 相手の番でも盤面のカードはクリックできるが、責任者以外の操作はエンジン側で無視されるため**盤面は壊れない**（UI上は「相手の番です」と表示）。
- 通信が不安定な環境では手の反映に遅延が出る（ターン制なので致命的ではない）。
